# Portfolio Publish Tool

> GitHub URLを渡すだけで、サニタイズ→Public化→Notion登録→X告知文生成までを自動化するローカルCLI

## 🎯 ポートフォリオ運用を月15分→3分に短縮（自分用に開発、5件公開で実証）



## Overview 概要

毎月1本ペースで作るアプリを、GitHubのPrivateリポジトリURLを起点にNotionポートフォリオへ自動公開するローカルCLIツール。原本Privateリポジトリは絶対に書き換えず、機密情報をサニタイズした複製を新規Public GitHubに発行し、Notion DBへの2層構造の子ページ作成、X告知文3パターンの生成までを1コマンドで完結させる。

Node.js 22+のネイティブテストランナーと外部依存を最小化したESMモジュール構成で実装。Notion統合トークンとgh CLI認証だけで動き、ローカル完結型のため運用コストはゼロ。MVP（v0.1.0）として14モジュール・4テンプレート・15サニタイズルール・38ユニットテストを実装し、自分の案件でE2E動作を確認済み。


## Problem 何を解決するか

- 毎月の公開作業（README整形・サニタイズ・Notion登録・告知文作成）に15分以上かかり継続が苦痛だった
- Privateリポジトリに含まれる機密情報を手動で削るのは見落としリスクが高く心理的負荷が大きい
- 公開作業を雑にやると原本Privateリポジトリを誤って改変・削除してしまう恐れがある
- Notion DBへの登録・告知文作成までを毎回手作業でやると体裁がブレて運用が続かない

## Solution どうやって解決するか

「原本は読み取り専用・複製のみ加工」という原則を二重防御で担保し、サニタイズも「ルールベース置換（第1層）」と「独立実装の残存スキャン（第2層）」に分離して冗長化。1件でも機密が残れば即中断する設計で、誤公開を構造的に防ぐ。

publish.mjsを起点にクローン→サニタイズ→READMEテンプレ展開→Public化→Notion 2層ページ生成→X告知文3パターン生成までをDAG的に実行し、--dry-runや--on-duplicateで冪等性も担保。


## Features 主な機能

- GitHub URLからPrivateリポジトリを一時ディレクトリにクローンし、git履歴を破棄して再コミット
- 15パターンの辞書置換＋独立実装の残存スキャンで二重に機密情報をサニタイズ
- Public用READMEをテンプレートから自動生成し、新規Public GitHubリポジトリを作成
- Notion ProjectsDBへのレコード追加と2層構造の子ページ生成を自動化
- X告知文をティザー／詳細／技術解説の3パターンで自動生成し280字制限を考慮
- --dry-run / --on-duplicate / --rollback-on-error 等のオプションで安全な再実行を保証
